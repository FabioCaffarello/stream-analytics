## WEAKNESSES

### W1. Multi-Replica Stream Coherence (FIXED this session)
- **Root cause**: `stream_controller.odin` flagged `Protocol_Invalid` (displayed as "compat mismatch") on any server timestamp regression, even from normal multi-replica interleaving
- **Impact**: Persistent false DESYNC state with 2+ processor replicas
- **Fix applied**: 5s tolerance for timestamp regressions + auto-recovery when valid timestamps resume
- **Residual risk**: Need to validate tolerance threshold under production load patterns

### W2. Stream Picker UX (FIXED this session)
- **Root cause**: Each subscription channel created a separate stream view slot; picker showed all 30 slots instead of deduplicating by venue+symbol
- **Impact**: 25+ duplicate "binance:BTCUSDT:SPOT" entries in stream picker
- **Fix applied**: Deduplication by venue+symbol in the CONNECTED section
- **Residual**: Stream counter still shows slot count (1/6) not market count (1/1) — cosmetic

### W3. Compare Mode Candle Scroll/Zoom (FIXED this session)
- **Root cause**: `build_compare.odin` hardcoded `scroll_x[0]` and `zoom[0]` for all compare panes instead of per-pane `[ci]` indexing. Also used `cell_effective_tf_string(state, 0)` and `cell_effective_tf_ms(state, 0)` instead of `ci`.
- **Impact**: All candle compare panes shared scroll/zoom state; TF labels wrong.
- **Fix applied**: Changed all `[0]` references to `[ci]` for per-pane independence.

### W4. Higher-TF Orderbook Stale Data (FIXED this session)
- **Root cause**: `apply_set_timeframe_action` in `stream_views.odin` cleared candle/heatmap/VPVR stores on TF switch but did NOT clear `orderbook_store` or reset `orderbook_snapshot_seen` on the active slot.
- **Impact**: Stale L2 data from prior TF persisted; DOM/footprint referenced wrong book state.
- **Fix applied**: Added `slot.orderbook_store = {}` and `slot.orderbook_snapshot_seen = false` on TF switch, mirroring the reconnect path.

### W5. Security Gaps (Unchanged from v8)
- No TLS termination (plaintext WS in production)
- API keys stored in JSONC config (not vault/secrets)
- No RBAC or tenant isolation
- No rate limiting enforcement validation
- ADR-0020 (GitOps Secrets) in progress but not complete

### W6. Server Health Reporting
- Both server (8080) and store (8083) report `"status":"degraded"` with `"ws_connected":false`
- This is because the server's health endpoint checks for upstream exchange WS connections
- Consumer handles exchange connections, not the server — health check may be misleading
- **Severity**: P2 — operational confusion

## THREATS

### T1. Competitive Market Terminal Landscape
- TradingView: dominant retail + institutional presence
- Bloomberg Terminal: institutional standard
- Bookmap, Sierra Chart: niche depth-of-market competitors
- Risk: MR needs clear differentiation (multi-exchange, self-hosted, open architecture)

### T2. Exchange API Instability
- Exchange WebSocket APIs change without notice
- Rate limit policies evolve
- New exchanges require adapter development effort
- Mitigation: canonical normalization layer (ADR-0017) + per-exchange adapters

### T3. WASM/Odin Ecosystem Maturity
- Odin is niche — smaller community, fewer libraries
- WASM Canvas2D has performance ceiling vs WebGL/WebGPU
- Browser WASM debugging tools are limited
- Mitigation: platform abstraction layer means renderer can be swapped

### T4. Regulatory Compliance
- Market data redistribution licensing (exchange agreements)
- GDPR/data residency for EU clients
- Financial data retention requirements
- Not yet addressed — critical for institutional sales
